ROPINDER — INSTRUCCIONES DE ADMINISTRACIÓN
============================================

CÓMO ENTRAR COMO ADMIN
-----------------------
No existe un usuario/contraseña de admin separado — es tu misma cuenta normal
de Ropinder, con un rol especial. Nadie es admin por default (ni siquiera
las cuentas de prueba).

1. Registrate normalmente en /signup con tu email y contraseña, como
   cualquier usuario.
2. Desde tu compu, en la carpeta del proyecto, corré:

     npm run admin:promote -- tu-email@ejemplo.com

   Esto te da rol ADMIN en la base de datos. No se puede hacer desde la
   web por seguridad (evita que cualquiera se auto-declare admin).
3. Iniciá sesión normalmente en ropinder.vercel.app/login con ese email.
4. Andá a ropinder.vercel.app/admin — ahí vas a ver el panel de moderación.

Si alguna vez necesitás promover a otra persona (un moderador de
confianza), es el mismo comando con su email.

QUÉ PUEDE HACER EL ADMIN (/admin)
-----------------------------------
- Ver la cola de reportes (usuarios reportando otros usuarios o prendas).
- Resolver un reporte (marcarlo resuelto) o descartarlo.
- Suspender (banear) o reactivar una cuenta de usuario. Un usuario
  suspendido no puede iniciar sesión hasta que lo reactivés.
- Reembolsar al comprador de un match reportado: si el pago ya se le
  liberó al vendedor, el sistema le saca la plata de su saldo (balance)
  y se la devuelve al comprador automáticamente.

CÓMO FUNCIONA LA PLATA (custodia / escrow)
--------------------------------------------
1. El comprador paga dentro del chat del match ("Pagar") — la plata NO va
   directo al vendedor, queda retenida por Ropinder (transacción tipo
   ESCROW_HOLD, estado PENDING).
2. Cuando el comprador confirma que recibió la prenda ("Confirmar
   Recepción"), la plata se libera al saldo (balance) del vendedor
   (ESCROW_RELEASE).
3. Esa plata liberada queda retenida 48 horas antes de poder retirarse
   (evita que alguien cobre y desaparezca antes de que se resuelva un
   reclamo). Se puede ver el desglose "disponible" vs "en espera" en
   /profile.
4. Si algo sale mal y se reporta el match, vos como admin podés
   reembolsar al comprador desde /admin — si la plata ya se había
   liberado al vendedor, se la sacamos de su balance automáticamente.

IMPORTANTE — BASE DE DATOS DE PRODUCCIÓN
-------------------------------------------
La app en Vercel usa Turso (SQLite remoto), no el archivo local dev.db.
El archivo .env local con TURSO_DATABASE_URL hace que hasta corriendo
"npm run dev" en tu compu estés hablando con la base de PRODUCCIÓN. Ojo
con eso al probar cosas — no es un entorno de prueba aislado.

Si en algún momento se agregan columnas o tablas nuevas al
prisma/schema.prisma, hay que sincronizar Turso a mano corriendo el
script correspondiente en scripts/ (sync-admin-schema-turso.ts,
sync-escrow-schema-turso.ts, sync-rating-schema-turso.ts son ejemplos
ya usados). Son todos aditivos (ALTER TABLE / CREATE TABLE), no borran
nada.

PAGOS REALES (MercadoPago) — PENDIENTE
------------------------------------------
Hoy el "Pagar" del chat es un cobro simulado (no mueve plata real).
Para cobrar de verdad hace falta:
1. Crear una cuenta de vendedor/negocio en MercadoPago
   (https://www.mercadopago.com.ar).
2. Sacar el Access Token de producción desde el panel de desarrolladores.
3. Pasarme esas credenciales (como variables de entorno, nunca en el
   código) para conectar el checkout real.

IDEAS PARA SEGUIR SUMANDO
----------------------------
- Notificaciones push (match nuevo, mensaje nuevo, reporte resuelto).
- Favoritos / wishlist de prendas.
- "Modo tienda" para vendedores con muchas prendas (perfil tipo catálogo).
- Verificación de identidad / insignia de vendedor confiable (paga).
- Fee de retiro (cobrar un % cuando alguien saca su saldo).
- Comisión por transacción liberada desde la custodia (la forma más
  directa de generar ingresos, ya que el escrow ya existe).

FORMAS DE GANAR PLATA CON LA APP
------------------------------------
- Comisión sobre cada transacción liberada desde la custodia (%).
- Suscripción Premium (ya existe) — swipes ilimitados.
- Bump / destacado de prendas (ya existe) — pagar por visibilidad.
- Fee de retiro de saldo.
- Insignia de vendedor verificado (paga).
- Espacio publicitario para marcas/boutiques dentro del feed.
